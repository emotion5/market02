## 2026-07-20

**주문/결제를 서버(DB)에 저장 — 무통장입금 접수**

- **주문(결제)은 로그인 필요.** 무통장입금으로 접수되며 상태는 **입금대기**로 시작, 세금계산서 신청 시 함께 접수됨(입금확인 후 발행 대기)

- 마이페이지 **결제내역·세금계산서 내역·배송조회가 실제 저장된 상태**를 표시 — 이전의 "시간이 지나면 자동으로 배송완료되던 데모 동작"을 제거. 배송상태·운송장은 앞으로 **관리자가 처리한 값**을 보여줌
- 개발용 시드에서 주문·견적 목업(localStorage)은 제거

## 2026-07-19

**견적서를 서버(DB)에 저장 — 계정에 귀속, 기기 바뀌어도 유지**
- 그동안 견적서는 **브라우저(localStorage)에만** 저장돼 다른 기기·브라우저에서는 볼 수 없고 브라우저를 비우면 사라졌음 → 이제 **서버 DB에 계정별로 보관**되어 어디서 로그인해도 견적 내역이 그대로 보임
- **견적서 발행은 로그인 필요**(계정에 귀속되는 문서이므로). 비로그인 상태로 발행을 누르면 로그인 화면으로 안내
- 발행 시 **가격은 서버가 회원 등급 기준으로 다시 계산**해 저장 — 화면 조작으로 금액이 바뀐 견적이 저장되지 않도록 함
- **유효기한을 발행 시점에 고정(스냅샷)** — 이후 관리자가 유효기간 설정을 바꿔도 이미 발행된 견적서의 유효기한·만료표시는 발행 당시 그대로 유지
- 개발용 시드에서 견적서 목업(localStorage)은 제거(견적은 이제 실제 로그인 후 발행)

**사이트 설정 — 공급자·계좌·유효기간·고객센터를 관리자 화면에서 편집**
- 그동안 **코드에 하드코딩**돼 있던 값들(공급자 상호·대표·사업자번호·주소·연락처, 무통장입금 계좌, 견적서 유효기간, 고객센터 이메일·전화)을 **관리자 '사이트 설정' 화면에서 직접 편집**하도록 변경 — 값 하나 바꾸는 데 개발·배포가 필요했던 것을 없앰
- 저장하면 **견적서·체크아웃(무통장입금 안내)·계정찾기** 화면에 즉시 반영됨. 견적서 유효기간을 바꾸면 견적서의 유효기한 표기·만료 판정에도 함께 적용
- 최초 값은 기존 값 그대로 옮겨와 **화면 변화 없이** 시작 (실제 사업자 정보로 교체는 이 화면에서)

**관리자 — 회원 등급·정지 기능 추가**
- **회원 등급** 도입: 일반(소비자가) / 회원도매가 2단계. 상품 상세에서 회원도매가가 보이는 자격을 **등급으로 관리** — 승인 사업자는 승인 시 자동으로 회원도매가 등급이 되고, 관리자가 **개인회원에게도 도매가를 부여하거나, 특정 회원의 도매가를 회수**할 수 있음
- **회원 정지/해제**: 활성 회원을 정지하면 **로그인·이용이 차단**되고, 해제하면 원래대로 복구(등급도 유지). 정지 시 사유·처리일이 기록되어 상세에서 확인 가능
- 회원 목록에 **'정지' 필터**와 도매 등급 표시, 상세에 **등급 뱃지·관리 버튼**(등급 부여/회수·정지/해제) 추가
- ※ 도매가 판정 기준을 '승인 사업자'에서 '**회원도매가 등급 + 활성 상태**'로 확장 — 기존 승인 사업자는 자동 이관되어 가격 변화 없음

**회원도매가 노출 정책 정리 — 승인 사업자에게만 표시**
- 이전에는 상품 상세에서 **비회원·개인회원에게도 '회원도매가' 줄이 그대로 노출**되었고, 그 옆 '소비자가'는 실제 값이 아니라 도매가에서 **15%를 역산한 임시 숫자**였음(개발용 mock)
- 이제 **소비자가는 실제 등록값**을 그대로 쓰고, **회원도매가 줄은 승인 사업자(로그인)에게만** 표시. 비회원·개인회원은 소비자가만 봄
- 실제로 장바구니·견적에 담기는 금액은 이전에도 자격에 맞게(승인 사업자=도매가) 정확히 계산되고 있었음 — 이번 변경은 **화면 표시**를 정책에 맞춘 것
- 임시 역산 로직(`getConsumerPrice`) 제거

**관리자 — 회원 관리 확장 (전체 목록·검색·상세)**
- 기존 '회원 승인'(사업자 승인 대기만 보이던 화면)을 **'회원 관리'로 확장** — 개인·사업자 **전체 회원**을 조회
- **검색**(이메일·상호·사업자등록번호)과 **필터**(유형: 개인/사업자, 상태: 활성/승인대기/반려/탈퇴), **페이지네이션**(20명 단위) 추가
- **회원 상세 페이지** 신설 — 이름·연락처·가입일, 사업자 정보(등록번호·상호·대표자·등록증·승인 이력·반려 사유)를 한 화면에서 확인. 승인 대기 사업자는 상세/목록 양쪽에서 바로 승인·반려 가능
- ※ 회원 **정지/해제**와 **등급** 기능은 저장 항목(DB) 추가가 필요해 다음 단계로 분리

**관리자 — 홈 노출 편성(진열대) 기능 추가**
- 관리자 상품 관리에 **'홈 노출 편성'** 화면 신설 — 메인 화면 카테고리별 진열대에 **어떤 상품을 어떤 순서로** 보여줄지 클릭으로 편성. 위로 올릴수록 앞에 진열되며 **카테고리당 최대 8개**
- 이전에는 이 진열 목록이 **파일(featured.json)에 고정**돼 있어 바꾸려면 개발자가 손대야 했음 — 이제 관리자가 직접 편성·저장
- 상품 목록 상단에 **'홈 노출 편성' 버튼** 추가
- **비활성(숨김) 상품은 홈 진열에서 자동 제외** — 진열에 넣어둔 상품을 나중에 비활성으로 바꾸면 홈에서 사라짐(편성 화면에는 '비활성' 표시와 함께 남아 정리 가능)

- 상품 상세 **다중 이미지 갤러리** 추가 — 메인 이미지 아래 썸네일 1행(최대 6장), 클릭 시 전환. 파일명 규칙(`상품id-1`, `-2`…)으로 자동 인식
- `faucet-0001` 상품에 상세 이미지 5장 추가
- 옵션이 하나뿐인 **단독 상품**은 옵션 선택 없이 소비자가·회원도매가·수량만 바로 표시

**상품 목록 · 카드**
- 카드 썸네일 우측 상단에 **견적서 바로 추가 아이콘**(+툴팁) 추가 — 상세에 안 들어가도 바로 담김
- 카드 **테두리 제거**, 마우스오버 시 **그림자로 떠오르는** 효과로 변경. 추가 아이콘도 그림자 스타일로 통일
- 상품 그리드 **열 수 자동 조정** + 페이지 최대폭 확대(1200→1600px) — 맥북에어에서 4열 노출
- 홈 카테고리 섹션을 좌측 카테고리명 + 우측 그리드 **2단 레이아웃**으로 변경

**내비게이션 · 레이아웃**
- 상단 카테고리 내비 개편 — '전체' 칩 제거, **흰색 알약(pill)형 바**, 마우스오버 시 **영문↔한글 전환**
- 오른쪽 콘텐츠 영역 **흐린 회색 배경**, 메인 최상단 대형 **Products** 타이틀 추가
- 브랜드 심볼을 **파비콘**(주소창 아이콘)으로 적용

**회원 · 로그인**
- 회원가입을 **개인/사업자 유형 선택** 구조로 분리 (`/signup` → `/signup/personal`, `/signup/business`), 헤더 버튼 '회원가입'으로 변경
- **이메일을 공용 계정 아이디(primary key)** 로 정리 — 로그인은 이메일 단일 입력, 사업자 가입에도 이메일 추가(사업자등록번호는 확인용)
- 개인회원 가입은 이메일·비밀번호만 (이름은 주문 단계에서 수집)
