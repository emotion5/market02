# 기능 정리 (완성 기능 한눈에)

> 날짜와 무관하게 **기능별로** 묶은 요약입니다.
> 날짜순 이력이 필요하면 `작업이력.md`를 보세요.
> (사용자에게 보이는 변화 중심 — 기술 상세는 git log 참고)


## 1. 상품 · 카탈로그 (고객 화면)

- 상품 목록: 카테고리별 진열, 목록형/카드형 뷰 전환, 상품 요약 표시
- 상품 카드: 테두리 없이 마우스오버 시 그림자로 떠오르는 스타일, 카드에서 바로 견적서에 담는 '+' 아이콘
- 그리드: 화면 폭에 따라 열 수 자동 조정(맥북에어 4열), 카테고리명 + 그리드 2단 레이아웃
- 상품 상세: 모달(팝업)로 열림, 직접 URL 접근 시엔 일반 페이지(공유·SEO 유지)
  - 다중 이미지 갤러리(썸네일 최대 6장), 색상 옵션 스와치
  - 배송·상세설명·크기 아코디언(+/− 펼침), 내용이 길면 정보 영역만 내부 스크롤
  - 수량 직접 입력, 옵션이 하나뿐인 단독 상품은 옵션 선택 없이 바로 표시
- 카테고리 내비게이션: 상단 알약(pill)형 바, 한글/영문 병기
- 홈: 롤링 배너, 카테고리별 진열대(관리자가 편성한 순서·상품)
- 검색 기능


## 2. 가격 정책 (소비자가 / 회원도매가)

- 2단 가격 체계: 소비자가(일반) / 회원도매가
- 회원도매가는 **승인 사업자(또는 도매 등급 회원)에게만** 상품 상세에 노출, 비회원·개인회원은 소비자가만 봄
- 장바구니·견적·주문 금액은 항상 **서버가 회원 등급 기준으로 재계산**해 저장(화면 조작 방지)


## 3. 견적서 (Quote)

- 견적 장바구니: 상품을 담아 견적 구성, 모바일 바텀시트 UX
- 견적서 발행: '견적서 발행' 버튼 → 그 시점 내용으로 **견적번호·발행일 확정**되어 저장(발행본은 읽기 전용)
- 계정 귀속 저장: 견적서를 **서버 DB에 계정별로 보관** — 기기·브라우저가 바뀌어도 유지 (발행은 로그인 필요)
- 유효기간 스냅샷: 발행 시점의 유효기한을 고정 — 이후 관리자가 설정을 바꿔도 발행본은 그대로
- 유효기간 지난 견적서는 '유효기간 만료'로 표시, 인쇄용 견적서 페이지 제공


## 4. 주문 · 결제 (무통장입금)

- 무통장입금 체크아웃, 주문은 로그인 필요
- 주문을 **서버 DB에 계정별로 저장** — 상태는 '입금대기'로 시작
- 포트원 가상계좌 자동 입금확인:
  - 주문별 전용 가상계좌 발급(포트원 V2, 토스페이먼츠 채널)
  - 고객이 입금하면 웹훅 → 서명검증 → 금액대조 → **자동 입금확인 처리**(관리자 수동 확인은 안전망)
  - 입금기한(24시간) 경과·결제실패 시 주문 자동 취소
  - 완료 화면에 발급 계좌번호·입금기한 표시
- 세금계산서 신청: 주문 시 함께 접수(입금확인 후 발행 대기)


## 5. 회원 · 인증 (고객)

- 회원가입: 개인/사업자 유형 분리 (`/signup/personal`, `/signup/business`)
- 이메일을 공용 계정 아이디(primary key)로 사용 — 로그인은 이메일 단일 입력
- 사업자 가입: 사업자등록증 파일 실제 업로드(비공개 저장소 보관, 관리자만 만료 링크로 열람)
- 로그인/로그아웃, 세션 쿠키
- 마이페이지: 프로필 편집·회원탈퇴, 결제내역·세금계산서 내역·배송조회, 견적서 내역
  - 배송상태·운송장·세금계산서 발행 여부는 **관리자 처리 결과를 그대로** 반영


## 6. 관리자 (Admin)

- 대시보드: 처리 대기 카드(사업자 승인·입금·배송·세금계산서), 참고 카드(신규 가입·유효 견적 수)
- 회원 관리: 전체 회원 목록·검색(이메일/상호/사업자번호)·필터·페이지네이션, 회원 상세
  - 사업자 승인/반려, 회원 등급 부여/회수(도매가 자격), 회원 정지/해제
- 상품 관리: 목록, 상품 등록·수정, 변형(색상) 편집, 이미지 업로드
  - 홈 노출 편성: 카테고리별 진열 상품·순서 편성(카테고리당 최대 8개, 비활성 상품 자동 제외)
- 카테고리 노출: 카테고리별로 상단 내비 노출·메인(홈) 진열대 노출을 각각 토글(독립). "내비는 끄고 홈에만" 같은 조합 가능
- 주문 관리: 목록·필터·검색, 입금확인 → 배송준비 → 배송시작(택배사·운송장) → 배송완료, 세금계산서 발행완료 처리
- 견적서 관리: 발행 견적 조회(검색·유효/만료 필터·상세, 읽기 전용)
- 사이트 설정: 공급자 정보·입금 계좌·견적 유효기간·고객센터 연락처를 화면에서 직접 편집(즉시 반영)


## 7. 백엔드 · 인프라

- Next.js 기반, Prisma + PostgreSQL로 데이터 영속화
- 인증/세션 백엔드, 서버 프록시 계층
- S3 호환 스토리지(Supabase Storage) — 상품 이미지·사업자등록증 등 파일
- 사이트 설정 DB화(하드코딩 값 제거), 재실행 가능한 샘플 데이터 시더


## 8. 디자인 · UI 톤

- 타이포그래피: Pretendard 자체 호스팅, 글자 굵기 정리(대형 Products 타이틀만 강조)
- 헤더 개편: 헤더 바 해체, 로고·아이콘을 셸 격자에 직접 배치, 프로스티드 글라스 카테고리 내비
- 레이아웃: 왼쪽 견적서 열(흰색) + 오른쪽 콘텐츠(회색) 2단 배경, 스크롤 시 헤더 사라짐
- 아이콘/카드/견적 패널 톤 정리(얇은 선, 옅은 그림자·헤어라인), 브랜드 로고·파비콘 적용
