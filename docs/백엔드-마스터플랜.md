# MMM MARKET — 백엔드 마스터플랜 (한 장 요약)

쇼핑몰 + **어드민**을 아우르는 phase별 로드맵. 상세 설계는 [백엔드-플랜.md](./백엔드-플랜.md).

## 구조 한 줄
한 프로젝트(`market02`) 안에서 **쇼핑몰(`app/`) + 어드민(`app/admin`) + 얇은 API(`app/api`) + 고립된 백엔드(`server/`)**.
화면은 DB 직접호출 금지 → 로직은 전부 `server/` → `server/`는 Next 비의존(나중에 분리 가능).
스택: **PostgreSQL · Prisma 7 · zod · argon2+jose 세션 · Neon(개발) · PG는 추상화 뒤**.

상태: ✅ 완료 · 🔨 진행중 · ⬜ 예정

> **핵심 의존성**: 어드민 화면들은 **ADMIN 세션**으로 접근하므로 **Phase 2(인증)가 먼저** 되어야 실제로 열립니다.
> 그래서 순서 권장: 카탈로그 읽기(완료) → **Phase 2 인증** → 그 위에 상품관리·사업자승인 등 어드민 → 이후 주문/결제.

## 어드민 사이트 구성 (`app/admin`)
- **공통**: 레이아웃 + 좌측 메뉴, `proxy.ts`가 `role=ADMIN`만 통과, 각 API에서 재확인.
- **상품관리**(P1): 상품·옵션·색상·이미지·featured 편집 → JSON 손편집 제거.
- **회원관리**(P2): 사업자 가입 승인/반려(사업자등록증 확인), 회원도매가 적용.
- **주문관리**(P5): 입금확인·배송준비/중/완료 상태 변경, 운송장 입력.
- **세금계산서**(P6): 신청 건 발행완료 처리.
- **(P7)** 결제·환불 내역 조회.

## 관통 원칙 (전 phase 공통)
- **스냅샷 불변**: 견적서·주문은 발행/주문 시점 값을 복제 저장 → 이후 가격 변경에도 과거 문서 불변.
- **금액**: 표시가=부가세 포함, 공급가액=round(합계/1.1). 서버가 최종 확정(클라 신뢰 금지).
- **교체 이음새**: 기존 `lib/*.ts` 함수 시그니처 유지, 내부만 `server/*`로 교체 → 페이지 무수정.
- **PG 추상화**: `server/payments/providers/*` 뒤에 토스/포트원 → 나중에 교체·병행 가능.
