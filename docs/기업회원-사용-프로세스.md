# MMM MARKET — 기업회원 사용 프로세스

인테리어 자재 B2B 스토어 **MMM MARKET**을 기업 사용자가 이용하는 전체 흐름을 정리한 문서입니다.
(가입 → 승인 → 상품 탐색 → 견적 → 견적서 발행 → 주문/결제 → 배송 추적 → 사후 관리)

> 현재 단계는 프론트엔드 UI 구현 단계입니다. 인증·결제·배송은 실제 백엔드 연동 전이며,
> 주문/견적/배송지 등의 데이터는 브라우저 `localStorage`에 저장되어 동작을 시연합니다.

> **회원 유형은 개인 / 사업자 두 가지**입니다. 이 문서는 사업자회원 흐름을 기준으로 설명하며,
> 개인회원은 서류·승인 없이 즉시 가입한다는 점만 다릅니다(가입 이후 탐색·견적·주문 흐름은 동일).

---

## 한눈에 보는 전체 흐름

```
[회원가입 → 사업자회원] → (영업일 1일 내 승인) → [로그인]
      → [상품 탐색] → [견적서에 담기] → [견적서 확인 · 인쇄/PDF · 발행]
      → [주문 / 결제(무통장입금)] → [입금] → [배송 추적]
      → [세금계산서 발행] → [마이페이지에서 사후 관리]
```

---

## 1. 회원가입

- **경로:** 헤더 우측 **"회원가입"** 버튼 → `/signup` → **회원 유형 선택**
  - **개인회원** → `/signup/personal` — 이메일로 **즉시 가입**(서류·승인 없음)
  - **사업자회원** → `/signup/business` — 사업자등록증 확인 후 **승인**, 회원도매가 적용
- **사업자회원 입력 항목**
  - **이메일** — 로그인 **아이디이자 계정의 primary key**입니다. (`example@email.com` 형식)
  - **비밀번호**
  - **사업자등록번호** — **사업자 확인용**입니다. `000-00-00000` 형식으로 자동 하이픈 처리되며 10자리를 채워야 합니다. (아이디가 아님)
  - **사업자등록증** — 이미지 또는 PDF 파일 업로드
- **승인 안내:** 가입 신청 버튼 위에 *"자료 입력 후 영업일 1일 이내에 승인됩니다"* 문구가 강조 표시됩니다.
- **신청 완료:** 접수 완료 화면으로 전환되어 승인 소요(영업일 1일)를 다시 안내합니다.

> **아이디는 왜 이메일인가요?**
> 개인·사업자 구분 없이 **이메일 하나를 공용 아이디**로 사용해 로그인·연락 수단을 단일화했습니다.
> 사업자등록번호는 로그인 아이디가 아니라 **사업자 여부를 확인하는 자료**로만 쓰이며, 제출하는 사업자등록증과 대조됩니다.
> (주민등록번호는 아이디로 사용하지 않습니다 — 법적 제한.)

## 2. 로그인 / 계정 찾기

- **로그인:** 헤더 우측 사람 아이콘 → `/login`
  - **이메일 + 비밀번호**로 로그인 (개인·사업자 동일)
- **아이디·비밀번호 분실:** 로그인 화면 하단 **"아이디 · 비밀번호 찾기"** → `/find-account`
  - 고객센터(이메일/전화) 문의 안내 페이지입니다. 문의 시 **상호(사업자회원) 또는 가입 이메일**을 함께 알려주면 본인 확인이 빠릅니다.
  - 비밀번호는 원본을 복구할 수 없으므로(암호화 저장), 재설정 방식으로 처리합니다.

## 3. 상품 탐색

- **홈:** `/` — **카테고리별 큐레이션 섹션**이 표시됩니다(`data/featured.json` 순서 기준).
- **전체 상품:** `/products` — 전체 상품 목록.
- **카테고리:** 하드웨어 / 가구재 / 바닥재 / 장판 / 위생 / 건자재 (`/category/[slug]`)
  - slug ↔ 표시 이름은 `lib/constants.ts`에서 관리합니다.
    (`hardware`=하드웨어, `panel`=가구재, `pressbevel`=바닥재, `antipress`=장판, `bathmatch`=위생, `stablecore`=건자재)
- **검색:** 헤더 검색창 → `/search?q=...`
- **보기 전환:** 목록 상단 토글로 **카드 보기 ↔ 리스트 보기** 전환
  - 카드에는 한 줄 요약, 색상 옵션, "오늘출발" 배송 배지가 표시됩니다.
- **상품 상세:** 상품 클릭 → 목록에서는 **상세 모달**로 열리고(인터셉트 라우트), 새로고침·직접 접근 시 `/products/[id]` 페이지로 열립니다. 옵션·색상 선택 후 견적서에 담습니다.

---

## 4. 견적서에 담기 (견적 스택)

- 상품 목록/상세 페이지 **좌측에 항상 견적서 패널**이 표시됩니다.
- 상품을 담으면 견적 스택에 쌓이고, 수량 조절·삭제·전체 비우기가 가능합니다.
- 헤더의 장바구니 아이콘에 담긴 개수가 배지로 표시됩니다.

> 이 견적 스택(장바구니)은 **작업 중인 임시 목록**입니다. 주문이 완료되면 자동으로 비워집니다(→ 8번 참고).

---

## 5. 견적서 확인 · 인쇄 / PDF · 발행

- **경로:** 견적 패널의 **"견적서 보기"** → `/quote`
- **종이 견적서(A4) 형태**로 표시됩니다.
  - 상단: 견적번호 · 견적일자 · **유효기간(발행일로부터 14일, `QUOTE_VALID_DAYS`)**
  - **공급받는 자(고객)**: 상호·담당자·연락처를 직접 입력
  - **공급자(MMM MARKET)**: 상호·대표자·사업자등록번호·주소·연락처
  - 합계금액을 한글 표기("일금 …원정")로 강조
  - 품목 표: 품목 · 규격 · 수량 · 단가 · 금액, 하단에 **공급가액 / 부가세(10%) / 합계금액** 자동 분리
- **인쇄 / PDF 저장:** 우측 상단 **"인쇄 / PDF 저장"** 버튼 → 브라우저 인쇄 대화상자에서 프린터 출력 또는 **PDF로 저장**
  - 인쇄 시 화면 장식(헤더/푸터/버튼)은 자동으로 숨겨지고 용지만 깔끔하게 출력됩니다.
- **견적서 발행:** `/quote`에 보이는 견적번호·발행일은 아직 **확정 전 미리보기**입니다.
  **"견적서 발행"**을 누르면 **그 시점의 번호·날짜·품목·가격이 그대로 고정**되어 저장되고, 발행된 견적서 페이지 `/quotes/[number]`로 이동합니다.
  - 발행된 견적서는 **읽기 전용**입니다. 내용을 바꾸려면 상품을 다시 담아 **새로 발행**해야 합니다.
  - 발행 이력은 **마이페이지 → 견적서 내역**(`/mypage/quotes`)에서 유효기간과 함께 다시 열람할 수 있습니다.

> **가격은 언제 기준인가요?**
> 견적서에는 유효기간(발행일로부터 14일)이 있으며, 주문이 확정되면 **주문 시점 가격이 스냅샷으로 고정**됩니다(→ 8번).

## 6. 주문 / 결제 (무통장입금)

- **경로:** 견적서 하단 **"주문 / 결제하기"** → `/checkout`
- **입력·확인 항목**
  1. **주문자·배송 정보** — 받는 분, 연락처, 배송지, 배송 메모
  2. **주문 상품** — 담은 품목·옵션·수량·금액 (배송비는 품목·수량·지역에 따라 별도 안내)
  3. **결제수단 = 무통장입금(사업자 거래)**
     - 입금 은행 · 계좌번호 · 예금주 + **공급자 사업자등록번호**를 다시 확인
     - **입금자명** 입력
  4. **세금계산서** — 전자세금계산서 발행 선택 시 **사업자등록번호(10자리)** · 상호 입력
  5. **결제 금액** — 공급가액 / 부가세(10%) / 배송비(별도) / 합계
- 필수 항목을 모두 채우면 **"주문하기"** 버튼이 활성화됩니다.

---

## 7. 주문 완료 & 견적 스택 처리

- 주문하기 → **주문 접수 완료** 화면
  - 주문번호 + **입금 계좌 · 입금 금액** 재안내
  - (세금계산서 신청 시) 입금 확인 후 입력한 사업자등록번호로 발행됨을 안내
- **이 시점에 처리되는 것**
  - 주문이 **히스토리(주문 스냅샷)로 영구 저장** → 마이페이지 결제내역에 표시
  - **견적 스택(장바구니)은 자동으로 비워집니다** — 이미 주문된 상품이 남아있을 필요가 없기 때문
  - (발행해 둔 견적서는 별도 문서라 그대로 남습니다 → `/mypage/quotes`)
- **"확인"** → 홈으로 / **"주문내역 보기"** → 마이페이지 결제내역

> **가격이 바뀌면 과거 주문 금액도 바뀌나요? — 아니요.**
> 주문 상세는 주문 시점의 상품명·단가·수량을 **그대로 복제(스냅샷)** 해 저장합니다.
> 이후 상품 카탈로그 가격이 변경되어도 **과거 주문·세금계산서 금액은 불변**입니다.
> (회계·세금·법적 증빙 정합성을 위한 커머스 표준 방식)

## 8. 마이페이지 — 사후 관리

- **진입:** 헤더의 마이페이지 아이콘 → `/mypage` (기본으로 결제내역으로 이동)
- **좌측 메뉴 6가지**

| 메뉴 | 경로 | 내용 |
|------|------|------|
| **견적서 내역** | `/mypage/quotes` | 발행한 견적서 목록. 견적번호·발행일·금액 + **유효기간**(만료 표시) |
| **결제내역** | `/mypage/orders` | 견적(주문) 단위 카드. 주문번호·일시·품목·금액 + **상태 뱃지** + **배송조회** |
| **세금계산서 내역** | `/mypage/tax-invoices` | 세금계산서 신청 주문의 사업자번호·공급가액·세액·합계·발행상태 |
| **배송지 관리** | `/mypage/addresses` | 자주 쓰는 배송지 등록/삭제/기본배송지 설정 |
| **회원정보 수정** | `/mypage/profile` | 상호·담당자·연락처·이메일·비밀번호 변경 |
| **회원 탈퇴** | `/mypage/withdraw` | 유의사항 확인 후 탈퇴 |

### 8-1. 배송 추적

- 결제내역 카드의 **"배송조회"** 버튼 → 택배사·운송장번호 + **5단계 타임라인**
- 진행 단계: **입금대기 → 입금확인 → 배송준비 → 배송중 → 배송완료**

### 8-2. 세금계산서

- 주문 시 전자세금계산서 발행을 신청한 건이 표시됩니다.
- 입금 확인(입금확인 단계) 이후 **발행완료**로 표시됩니다.

## 참고 — 현재 구현 상태(개발 노트)

- **인증/세션:** 로그인·가입은 화면(UI)만 구현. 실제 인증 서버 연동 전. 로그인 상태는 이메일 기준으로 `localStorage`에 보관.
- **로컬 저장 데이터:** 브라우저 `localStorage`에 저장
  - `market02-cart` (견적 스택) · `market02-quotes` (발행 견적서) · `market02-orders` (주문) · `market02-addresses` (배송지) · `market02-profile` (회원정보) · `market02-auth` (로그인 상태)
- **배송상태:** 백엔드가 없어 **주문 후 경과 시간에 따라 자동 진행되는 데모용 모의 상태**입니다(대략 1분 후 입금확인 → 2분 배송준비 → 4분 배송중 → 7분 배송완료).
- **교체 필요한 자리표시값(TODO):** `lib/constants.ts`의 공급자 정보(`SUPPLIER`)·입금 계좌(`BANK_ACCOUNT`)·유효기간(`QUOTE_VALID_DAYS`), `/find-account`의 고객센터 연락처.
- **회원 탈퇴 시** 위 로컬 데이터를 정리합니다(실서비스에서는 법령상 보관 의무 자료는 별도 보관).
